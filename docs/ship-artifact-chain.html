              Đây là checklist human-friendly cho những gì mỗi platform delta thường nên chứa.

              Web UI + flow

              Web delta nên biến feature đã approve thành một kế hoạch UI và state bám sát working repo web.

                Android screen + architecture

                Android delta nên giải thích feature gắn vào app architecture, navigation và state layers hiện tại như thế nào.

                  iOS screen + ownership

                  iOS delta nên nối feature với đúng ownership boundary trong app.

                    Backend contract + behavior

                    Backend delta nên mô tả thay đổi hành vi phía server, không chỉ liệt kê endpoint.

                      QE automation scope

                      QE delta nên chỉ rõ automation nào cần thêm hoặc sửa, và automation repo đang có constraint gì.

                        QC manual quality

                        QC delta nên là manual-quality planning trong spec repo, không phải working-repo implementation handoff.

                          Một delta tốt thường trả lời được những câu hỏi này
